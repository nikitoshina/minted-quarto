\usemintedstyle{bw}

This filter adds minted and listings environments to Quarto. It is based
on the \href{https://ctan.org/pkg/minted}{minted} package for LaTeX. The
minted package uses the Python Pygments library to highlight code. Since
the minted package is not included in the default LaTeX distribution,
you will need to install it separately. Please refer to the
\href{https://ctan.org/pkg/minted}{minted documentation} for
installation instructions.

\begin{listing}[H]
\begin{minted}{r}
print("hello")
\end{minted}
\end{listing}

\begin{verbatim}
[1] "hello"
\end{verbatim}

To use the minted-quarto filter to compile into PDF, include the
following in the \mintinline{text}{YAML} front matter:

\begin{listing}[H]
\begin{minted}{yaml}
pdf-engine-opts:
    - -shell-escape
\end{minted}
\end{listing}

You can specify whether you want to transform inline code blocks into
minted environments; by default, this is set to true.

You can also pass any options to the minted package. Although this
feature might not be useful in most cases, it can be handy in specific
situations. I recommend using this filter in workflows where you have
Quarto render LaTeX that you will later edit further.

Additionally, you can specify the level at which to create listing
environments. By default, this is set to ``chapter,'' but it can be
changed to ``section.'' Refer to the
\href{https://ctan.org/pkg/minted}{minted documentation} for more
information.

Code block level options are not supported.

To set the style, refer to the documentation and include the declaration
as follows:

\begin{quote}
Unfortunately, include-in-header places it before minted is loaded, so
we have to do it this way :(
\end{quote}

\begin{listing}[H]
\begin{minted}{yaml}
include-in-header:
      - text: |
            \usemintedstyle{name}
\end{minted}
\end{listing}
